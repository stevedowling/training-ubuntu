# Module 7 · Networking & Storage

> **Goal:** diagnose "the network is down" and "the disk is full" like a professional - and understand how Linux sees network interfaces, disks, and filesystems.

## Learning objectives

- Inspect interfaces, addresses, and routes with `ip`; sockets with `ss`
- Debug connectivity layer by layer: `ping`, DNS (`resolvectl`, `dig`), `curl`
- Configure networking the Ubuntu way with netplan
- Use SSH fluently: keys, `ssh-copy-id`, `scp`/`rsync`, config file
- Understand block devices, partitions, and filesystems: `lsblk`, `mount`, `/etc/fstab`
- Find what's eating disk space (`df`, `du`, `ncdu`) and add a new disk end to end

## Planned lessons

1. The network stack - ip, ss, and debugging connectivity
2. SSH - remote work done right
3. Disks and filesystems - block devices, mounts, fstab
4. Space and growth - df, du, LVM basics
